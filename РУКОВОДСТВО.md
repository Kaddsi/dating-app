## 🎯 Что я создал для вас

Я разработал **полноценное приложение для знакомств** в Telegram (Telegram Mini App) с:

## 📁 Что входит в проект

### 1️⃣ BACKEND (Серверная часть на Python)

#### `phone_verification.py` - Проверка номера и блокировка регионов
**Что делает:**
- Заставляет пользователя поделиться номером телефона при первом запуске
- Проверяет код страны номера
- **БЛОКИРУЕТ** номера из:
  - 🇷🇺 России (+7)
  - 🇧🇾 Беларуси (+375)
  - 🇮🇷 Ирана (+98)
  - 🇮🇶 Ирака (+964)
- **Умная проверка**: отличает российские номера +7 от казахстанских по коду города
- Сохраняет данные в базу PostgreSQL

**Пример работы:**
```python
# Пользователь из России с номером +7 495 123-45-67
→ Система проверяет код 7495 (Москва)
→ Находит в списке российских кодов
→ БЛОКИРУЕТ и отправляет вежливое сообщение:
   "К сожалению, наш сервис пока недоступен в вашем регионе"

# Пользователь из Казахстана с номером +7 777 123-45-67  
→ Система проверяет код 7777
→ НЕ находит в российских кодах
→ РАЗРЕШАЕТ доступ
```

#### `main.py` - Основной бот
**Что делает:**
- Обрабатывает команду /start
- Показывает кнопку "📱 Поделиться номером"
- После проверки номера даёт доступ к веб-приложению
- Подключает middleware для проверки

#### `swipe_handler.py` - API для свайпов и мэтчей
**Что делает:**
- Обрабатывает свайпы (лайки/дизлайки)
- Автоматически определяет совпадения (когда два пользователя лайкнули друг друга)
- Отправляет уведомление в Telegram: "🎉 У вас мэтч!"
- Возвращает список потенциальных партнёров на основе расстояния

### 3️⃣ DATABASE (База данных PostgreSQL)

#### `schema.sql` - Структура базы данных
**Таблицы:**

**users** - Пользователи
- id, telegram_id, phone, country_code
- Профиль: имя, био, фото, интересы
- Местоположение (с поддержкой геопространственных запросов)
- Предпочтения: пол, возраст, расстояние

**swipes** - Свайпы
- from_user_id, to_user_id
- swipe_type: like, dislike, superlike

**matches** - Совпадения
- user1_id, user2_id
- Создаётся АВТОМАТИЧЕСКИ через триггер при взаимном лайке

**messages** - Сообщения (для будущего чата)

**Умная логика:**
```sql
-- Когда пользователь А лайкает пользователя Б:
1. Записывается свайп в таблицу swipes
2. Триггер проверяет: а лайкал ли Б пользователя А?
3. Если да → автоматически создаётся запись в matches
4. Backend видит мэтч и отправляет уведомления обоим
```

## 🔐 Безопасность

### Как работает защита от подделки

```python
# 1. Telegram отправляет initData с хэшем
initData = "user={...}&hash=abc123..."

# 2. Backend проверяет хэш
secret_key = HMAC(bot_token)
calculated_hash = HMAC(initData, secret_key)

# 3. Если хэши совпадают → данные настоящие
if calculated_hash == received_hash:
    ✅ Пользователь подтверждён
else:
    ❌ Попытка подделки
```

### Защита от спама
- Блокированные пользователи сохраняются в БД
- При повторной попытке → мгновенный отказ
- Не обрабатываются запросы без номера телефона

## ❓ Частые вопросы

**В: Как бот узнаёт, из какой страны пользователь?**
О: По номеру телефона. Каждая страна имеет свой код (+380 - Украина, +7 - Россия/Казахстан и т.д.)

**В: Почему блокируются именно эти страны?**
О: Это было в задании. Можно легко изменить список в коде.

**В: Можно ли добавить больше стран в блокировку?**
О: Да! Просто добавьте коды в список `BLACKLISTED_PREFIXES`:
```python
BLACKLISTED_PREFIXES = {
    "+7",    # Россия
    "+375",  # Беларусь
    "+98",   # Иран
    "+964",  # Ирак
    "+1",    # США (пример)
}
```

**В: Как работает определение расстояния?**
О: Используется PostGIS - расширение PostgreSQL для работы с геоданными. Оно считает расстояние между координатами пользователей.

**В: Нужен ли мне сервер?**
О: Да, нужен сервер для:
- Запуска Python бота
- Базы данных PostgreSQL
- API для фронтенда

Самый дешёвый вариант: DigitalOcean за ~$30/месяц

**В: Можно ли запустить бесплатно?**
О: Частично:
- Frontend на Vercel - бесплатно
- Backend можно на Railway (бесплатный тариф ограничен)
- База данных - нужно платить минимум $15/месяц
